The Problem

Manual Data Entry

Every Indian freelancer has a pile of vendor invoices. Keying GST numbers, 6-digit HSN codes, and splitting CGST/SGST into Tally or Excel manually is tedious and highly error-prone.

The Solution

Automated Extraction

InvoiceGhost handles standard GST invoices, non-GST receipts, foreign currency invoices, and even mixed-language (Hindi/English) formats. Drag, drop, export to CSV.

System Architecture

A utilitarian extraction engine built on FastAPI, secured by strict file validation and powered by dual LLM fallbacks.

01. AI Engine

Dual-Model Processing

Primary extraction via Gemini 2.0 Flash. If it fails, the system automatically falls back to Groq Llama 3.2 90B Vision for maximum reliability.

02. Authentication

Zero Account Policy

Login forms kill conversion. No user tables, no JWTs, no email verification. Just drop the file and get the data immediately.

03. Confidence

Heuristic Scoring

Returns a 0.0-1.0 confidence score per extraction. Low scores (< 0.8) automatically flag the UI to suggest human review for blurry scans.

04. Format

Mixed-Language Support

Seamlessly handles Indian invoice quirks, including English headers mixed with Hindi product descriptions without parsing failure.

05. Output

CSV Export Safe

One-click CSV generation with full line items and tax summary rows. Protected against CSV injection (neutralizes =,+,-,@ prefixes).

06. Testing

56/56 Tests Passing

Backend fortified with robust Pytest coverage spanning MIME spoofing rejection, confidence score clamping, and FastAPI rate limiting.

Developer API

Use the /api/parse endpoint to integrate InvoiceGhost directly into your own ERP, Tally workflow, or custom internal tools.

POST /api/parse

Uploads the invoice as multipart/form-data. Triggers strict MIME type and magic bytes validation before converting to a 300DPI image array for the extraction model.

Hardened Security

Designed with a security-first mindset for a tool handling sensitive financial documents.

MIME Spoofing Protection

Magic bytes validation ensures a file claiming to be a PDF is not actually an executable before processing.

EXIF Leakage Stripped

All embedded image metadata (location, device info) is aggressively stripped before transmission to external AI APIs.

Zero Data Retention

Files are processed entirely in memory. Temp files are deleted in strict `finally` blocks using `shutil.rmtree`.

Constant-Time HMAC

License keys for Batch Mode are validated using `hmac.compare_digest()` to prevent timing side-channel attacks.

Execution Modes

The free tier handles one file at a time. Unlock Batch Mode to process multi-file directories.

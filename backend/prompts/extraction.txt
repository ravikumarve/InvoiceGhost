2. **Language Support**: Handle mixed-language invoices (English + Hindi headers common in India). Extract data regardless of language.

   - Image quality (blurry, low resolution = lower score)
   - Text clarity and readability
   - Completeness of extracted data
   - Certainty of extracted values

## EXTRACTION GUIDELINES

### Invoice Identification
- **Invoice Number**: Look for labels like "Invoice No", "Invoice #", "Bill No", "बिल नंबर"
- **Invoice Date**: Extract in ISO format (YYYY-MM-DD). Handle formats like DD/MM/YYYY, DD-MM-YYYY, MM/DD/YYYY
- **Due Date**: Look for "Due Date", "Payment Due", "देय तारीख"

### Vendor & Buyer Information
- **Vendor Name**: The seller/issuer of the invoice. Look for "From", "Sold By", "विक्रेता"
- **Vendor GSTIN**: 15-character alphanumeric GST identification number
- **Buyer Name**: The recipient/customer. Look for "To", "Billed To", "खरीदार"
- **Buyer GSTIN**: 15-character alphanumeric GST identification number

### Line Items
- **Description**: Product/service name (required field)
- **HSN/SAC Code**: 6-8 digit Harmonized System/Nomenclature code for GST classification
- **Quantity**: Numeric value, handle decimals
- **Unit**: Unit of measurement (pcs, kg, liters, hrs, etc.)
- **Rate**: Price per unit
- **Amount**: Total for line item (quantity × rate)
- **Tax Rate**: GST percentage applied (e.g., 0, 5, 12, 18, 28)

### Tax Calculation (Indian GST)
- **CGST**: Central GST (half of total GST for intra-state transactions)
- **SGST**: State GST (half of total GST for intra-state transactions)
- **IGST**: Integrated GST (full GST for inter-state transactions)
- **Total Tax**: Sum of all taxes (CGST + SGST or IGST)
- **Subtotal**: Sum of all line item amounts before tax
- **Grand Total**: Subtotal + Total Tax

### Number Format Handling
- Handle Indian number formats with commas (e.g., "1,25,000.00" = 125000.00)
- Handle lakhs (e.g., "2.5 L" = 250000.00) and crores (e.g., "1.5 Cr" = 15000000.00)
- Extract numeric values only, ignore currency symbols (₹, Rs, INR)

### Date Format Handling
Convert all dates to ISO format (YYYY-MM-DD):
- DD/MM/YYYY → YYYY-MM-DD
- DD-MM-YYYY → YYYY-MM-DD
- MM/DD/YYYY → YYYY-MM-DD
- Month names (e.g., "15 Jan 2024" → "2024-01-15")

### Confidence Scoring Rubric
- **0.9-1.0**: High-quality image, clear text, all fields extracted with high certainty
- **0.7-0.9**: Good quality, minor blur, most fields clear, some uncertainty
- **0.5-0.7**: Moderate quality, some blur, several fields unclear or missing
- **0.3-0.5**: Poor quality, significant blur, many fields unclear or missing
- **0.0-0.3**: Very poor quality, barely readable, extraction highly uncertain

### Error Handling
- If image is too blurry or unreadable, set `confidence_score` < 0.3 and populate only clearly visible fields
- If invoice structure is non-standard, do your best to map available data to the schema
- If multiple invoices are present, extract data from the first/most prominent one
- If line items are not clearly separated, extract as best as you can from available data
